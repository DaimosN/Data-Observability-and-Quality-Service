# Data Quality Service

## 📋 Описание проекта

Сервис контроля качества данных (Data Quality Service) для кадровой системы, обеспечивающий валидацию, очистку и мониторинг данных о сотрудниках при загрузке из Excel-файлов. Система автоматически проверяет корректность данных, помещает проблемные записи в карантин и предоставляет comprehensive метрики для observability.

<u>Предмет исследования</u> — методы и инструменты мониторинга качества данных и Data Observability, включая валидацию, зону карантина и дашборды для выявления инцидентов. Фокус на конкретных механизмах для проактивного контроля.

<u>Цель исследования</u> — разработать и внедрить систему Data Observability с валидацией и карантином для повышения надежности кадровых данных, сокращения инцидентов и предотвращения загрузки некорректных записей в основную базу.

### Основные возможности

- **Загрузка и валидация Excel-файлов** с данными сотрудников
- **Многоуровневая система проверок** (форматные, бизнес-правила, справочные данные)
- **Карантин проблемных записей** с сохранением исходных данных и ошибок
- **Полноценный мониторинг** на базе Prometheus с 20+ метриками
- **Автоматическое определение качества данных** (completeness, uniqueness, freshness)
- **Обнаружение аномалий** и статистических выбросов

## ✅ Система валидации

### 1. Форматные валидаторы

| Валидатор | Описание | Пример ошибки |
|-----------|----------|---------------|
| **ФИО** | Проверка минимальной длины (≥2 символа), только буквы и дефис | `"Имя должно содержать минимум 2 символа"` |
| **Паспорт** | Серия (4 цифры), номер (6 цифр), только цифры | `"Серия паспорта должна содержать 4 цифры"` |
| **ИНН** | Проверка контрольной суммы для 12-значного ИНН | `"Неверный формат ИНН"` |
| **СНИЛС** | Проверка формата и контрольного числа | `"Неверный формат СНИЛС"` |
| **Телефон** | Российский формат (+7/8, 11 цифр) | `"Неверный формат телефона"` |
| **Email** | RFC 5322 compliant проверка | `"Неверный формат email"` |
| **Даты** | Множественные форматы (YYYY-MM-DD, DD.MM.YYYY, DD/MM/YYYY) | `"Неверный формат даты"` |

### 2. Бизнес-валидаторы

| Валидатор | Правило | Сообщение об ошибке |
|-----------|---------|---------------------|
| **Возраст** | 18-100 лет | `"Сотруднику {age} лет (минимум 18)"` |
| **Дата рождения** | Не в будущем | `"Дата рождения не может быть в будущем"` |
| **Дата приема** | Не в будущем, после даты рождения | `"Дата приёма не может быть раньше даты рождения"` |
| **Дата увольнения** | После даты приема, не в будущем | `"Дата увольнения не может быть раньше даты приёма"` |
| **Зарплата** | 20,000 - 5,000,000 руб | `"Зарплата ниже МРОТ"` / `"Зарплата превышает допустимый лимит"` |
| **Пол** | 'М' или 'Ж' | `"Пол должен быть 'М' или 'Ж'"` |

### 3. Справочные валидаторы (Reference Data)

| Валидатор | Источник | Кеширование |
|-----------|----------|-------------|
| **Должность** | `hr.dict_positions` | 5 минут |
| **Подразделение** | `hr.dict_departments` | Бессрочно |

## 📊 Система метрик (Observability)

### Метрики валидации (Counter)

| Метрика | Лейблы | Описание |
|---------|--------|----------|
| `dq_validations_total` | `status`, `source` | Общее количество валидаций |
| `dq_validation_errors_total` | `error_type`, `field` | Счетчик ошибок по типам и полям |
| `dq_files_processed_total` | `file_type`, `status` | Количество обработанных файлов |

**Пример использования:**
```promql
# Процент успешных валидаций
rate(dq_validations_total{status="approved"}[5m]) / rate(dq_validations_total[5m]) * 100
```


Метрики качества данных (Gauge)


| Метрика               | Лейблы                                   | Описание                        |
|-----------------------|------------------------------------------|---------------------------------|
| `dq_overall_score`    | `table_name`                             | 	Общая оценка качества (0-100)  |
| `dq_completeness`     | 	`table_name, column_name`               | 	Полнота данных по колонкам (%) |
| `dq_uniqueness`       | 	`table_name, column_name`               | 	Уникальность значений (%)      |
| `dq_freshness_hours`  | 	`table_name`                            | 	Часов с последнего обновления  |
| `dq_anomaly_count`    | 	`table_name, column_name, anomaly_type` | 	Количество аномалий            |
| `dq_duplicate_groups` | 	`table_name`                            | 	Количество групп дубликатов    |


Метрики состояния системы (Gauge)

| Метрика                          | Лейблы           | Описание                         |
|----------------------------------|------------------|----------------------------------|
| `dq_quarantine_size`             | `status`         | 	Записей в карантине по статусам |
| `dq_table_size_rows`             | `table_name`     | 	Общее количество записей        |
| `dq_last_load_timestamp_seconds` | `table_name`     | 	Timestamp последней загрузки    |
| `dq_salary_distribution`         | `salary_range`   | 	Распределение зарплат           |
| `dq_active_rules`                | `rule_category`  | 	Количество активных правил      |


Метрики производительности (Histogram)

| Метрика                               | Лейблы            | Buckets                                      | Описание                        |
|---------------------------------------|-------------------|----------------------------------------------|---------------------------------|
| `dq_validation_duration_seconds`      | `validation_type` | 0.01, 0.05, 0.1, 0.5, 1, 2, 5, 10, 30        | 	Время валидации                |
| `dq_batch_size_records`               | `source`          | 1, 5, 10, 25, 50, 100, 250, 500, 1000, 5000  | 	Размер обрабатываемых пакетов  |
